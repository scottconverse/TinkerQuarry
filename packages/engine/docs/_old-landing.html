Describe a part in plain words. Get a checked, print-ready file — on your own machine.

KimCad turns a description — or a photo, or a dimensioned sketch — into solid, parametric, 3D-printable geometry, validates it against your printer and material, and exports or sends it. The AI runs locally through Ollama, so your prompts, photos, and sketches never leave the computer. No CAD skills, no account, no cloud.

Quick start

Three steps to a print-ready part

No terminal, no account, no CAD. The first AI design takes roughly one to two minutes; template parts then re-render from sliders instantly.

1

Install & open

Download the installer, run it past SmartScreen (More info → Run anyway), and open KimCad. One Set up KimCad's AI button sets up the local AI engine (reusing a system Ollama, or downloading its own portable ~1.4 GB build) and fetches the two AI models (~7.7 GB) — all with a progress bar. No manual Ollama install.

2

Describe your part

Type what you want — or drop in a photo or a dimensioned sketch. KimCad asks at most one clarifying question, then builds checked, parametric geometry and shows a live 3D preview.

3

Export or print

Tune it with live sliders, then download an STL / editable .STEP, or — after an explicit confirmation — slice it and send it straight to your printer.

Architecture

How a description becomes a printable part

One deterministic pipeline, the same whether you drive it from the browser UI or the CLI. The geometry comes from a parametric template engine — not a neural net — so the output is closed, watertight, and dimensionally meaningful by construction.

01 · Input

Words, photo, or sketch

Plain English, a photo of a part, or a dimensioned sketch. Images are read by a local vision model and never leave the machine.

02 · Plan

AI design plan

The local planner turns your request into a structured design plan (JSON), asking one clarifying question only when a load-bearing dimension is missing.

03 · Geometry

Template / OpenSCAD

A matching parametric family (or generated OpenSCAD) renders to a real mesh with live, deterministic sliders.

04 · Check

Printability gate

Every part is validated against your printer and material, auto-oriented and hardened, and scored 0–100 for readiness.

✓ Must pass before slicing
05 · Deliver

Slice · export · send

Download the STL or editable .STEP, or slice to G-code and send to a printer — always behind an explicit confirmation.

Real printability, before you slice

The bundled PrintProof3D engine adds overhang, bridge, and bed-adhesion analysis on top of the deterministic gate, and writes a Smart Mesh readiness card — a 0–100 score, a plain verdict, the risks, and concrete recommendations. A part that fails the gate can never be sliced or sent — only downloaded to inspect.

86 parametric template families

Boxes, hooks, brackets and organizers; a decor world of frames, dishes, planters, ornaments and stands; and engineering hardware — washers, plates, brackets, standoffs, Gridfinity, fasteners. Each family carries an honest tier (benchmarked = what-you-set-is-what-you-get, or baseline = real geometry with a real-world fit caveat to verify) and a trusted CadQuery .STEP twin.

Local-first & private

Out of the box KimCad talks to a local runtime (Ollama or LM Studio) — no API key, no network, nothing leaves the machine. The core path is CPU-only; no discrete GPU required. A cloud model is an off-by-default opt-in: Settings → Cloud acceleration routes prompts (never images) through OpenRouter with your own key.

Direct send to your printer

A sliced job can go straight to a printer connection, always after an explicit confirmation. Supported connectors: Bambu (native LAN), OctoPrint, Moonraker, PrusaLink, Duet/RepRapFirmware, and Marlin (serial or network bridge). Each connector is validated against a conformance mock of the printer's real protocol.

Technical specs

What it runs on

PlatformWindows 10 / 11 — zero-terminal installer (the beta). macOS & Linux run from source (pip install, then kimcad web); code-verified cross-platform, not yet exercised on real mac/Linux hardware.
Local AI — planningqwen2.5:7b (~4.7 GB) via Ollama — the on-device planner that won the on-machine bake-off 4/4. Turns your words into a validated design plan.
Local AI — visionqwen2.5vl:3b (~3 GB) — the dedicated reader for photos and dimensioned sketches. Images are always processed locally and never persisted.
Runs offlineFully. No account, no API key, no network required after setup. KimCad even sets up its own AI engine on first run — reusing a system Ollama if present, else downloading Ollama's portable build (~1.4 GB) into its own data folder — so there's no manual install. The cloud path is an opt-in OpenRouter escape hatch for hard prompts only.
HardwareCPU-only core path (no discrete GPU). Reference box: a recent CPU with 32 GB RAM; the model advisor downshifts smaller machines.
Bundled enginesEmbedded CPython 3.13, OpenSCAD, OrcaSlicer (full ~65-brand / 1,400+ machine-profile library), and the PrintProof3D validation engine — each run at arm's length as a subprocess.
Printer catalogA curated ~29-printer catalog (Bambu, Creality, Prusa, Anycubic, Elegoo, Qidi, Sovol), each build-volume-gated and slice-proven in CI; three are reference printers also wired for native direct-send.
OutputsSTL / 3MF mesh, sliced G-code 3MF, and an editable .STEP (with the optional CadQuery engine) you can keep modeling in Fusion / FreeCAD / SolidWorks.
LicenseApache-2.0 core. Bundled GPL/AGPL engines are invoked as separate subprocesses, never linked.
For power users

There's a CLI too

The same pipeline, from the terminal. A bare prompt is treated as the design verb; --slice is the explicit print confirmation, and --send dispatches to a configured connector.

# Describe a part — writes a checked, validated model to output/ kimcad "a wall hook with two M4 screw holes 30 mm apart and a 35 mm arm" # Design, slice, and send to a printer (each flag is an explicit confirm) kimcad "a 40 mm cable clip" --printer bambu_a1 --material pla --slice kimcad design "a cable clip" --send octoprint # Launch the local browser UI (loopback-only by default) kimcad web # Pick a hardware-matched local model · run the 10-prompt done-gate kimcad models kimcad bench --min-success-rate 0.8

The browser UI is served at http://127.0.0.1:8765 and is a thin SPA over a documented local JSON API — see the API reference. An MCP server (python -m kimcad.mcp_server) exposes the printer as agent tools.

Honest beta status

What's proven — and what isn't yet

This is a beta, and we won't pretend otherwise.

All eleven build stages are complete and gate-passed. But real-hardware print validation is the beta's own job — every send-to-printer connector is validated against a runnable conformance mock of the printer's real protocol, not yet on physical metal (#11). If you have a printer, your report is the single most valuable thing you can give back.

The installer is unsigned (no code-signing certificate yet), so Windows SmartScreen will warn "Windows protected your PC" — that's expected; click More info → Run anyway. The install guide shows how to verify the SHA-256 checksums (SHA256SUMS.txt) and the build manifest (release-manifest.json, which records the exact source commit and is flagged unsigned_build) attached to each release.
