Engine running

TinkerQuarry's engine is running here. The user interface is not.

This port serves the engine's HTTP API (everything under /api/). The app you design parts in is built and shipped separately, so there is no interface to show you on this page. Here is how to get one.

Pick whichever fits

  1. You just want to use TinkerQuarry

    Close this tab and open the installed TinkerQuarry app from the Start menu. It starts its own copy of this engine and talks to it directly - you do not need to run kimcad web at all.

  2. You are developing the interface

    Leave this engine running and start the dev interface in a second terminal. It proxies its API calls to this engine, and it is always built from the current source: 

    cd path\to\TinkerQuarry\apps\ui
pnpm install
pnpm dev

    Then open http://localhost:1420 - the dev interface, not this page. If the engine rejects your requests, restart it with $env:TINKERQUARRY_DEV_TOKEN = "tq-dev-token" set, which is what the dev interface sends.

  3. You only wanted to check the engine is alive

    It is - you are reading a page it served. For a machine-readable answer, request /api/health.

Why there is no interface bundled here

There used to be. A pre-built copy of the app was committed next to the engine and served from this address, but nothing rebuilt it or checked it, so it silently fell eight pull requests behind the real source and started showing a setup wizard that no longer existed in the product. A page that is confidently wrong is worse than a page that tells you where to go, so the stale copy was deleted rather than patched.

TinkerQuarry engine · API docs live in the repository under packages/engine/docs/ · the dev workflow above is the one documented in docs/USER-MANUAL.md.
